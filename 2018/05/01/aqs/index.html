Java与JVM：（五）并发的范式——AQS

发表于 2018-05-01 | 分类于 Java

背景

JUC包中包括了各种基于乐观锁的并发工具，如可重入锁、读写锁、信号量等，这些并发工具存在一定的相似性，在某些场景下可以相互替换。JDK没有选择为每个并发工具独立开发其实现机制，而是在同一套模版上进行各自的实现，这套模版就是AQS。

AQS的全称是AbstractQueuedSynchronizer，该抽象类位于JUC包的locks下，它将并发操作抽象出两个最主要的步骤——获取资源和释放资源，将资源抽象为两种——独占资源（Exclusive）和共享资源（Share）。主要包括4种主要方法：acquire（获取独占资源）、acquireShared（获取共享资源）以及release（释放独占资源）、releaseShared（释放共享资源）。下文将对具体的实现机制进行阐述。

并发工具在实现时并不直接继承AQS，而是利用一个内部类来实现AQS，使用时操纵该内部类变量来实现线程安全。
