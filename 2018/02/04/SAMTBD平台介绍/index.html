SAMTBD平台介绍

发表于 2018-02-04

背景

SAMTBD——轨迹大数据分析与挖掘平台，是武汉大学遥感学院秦昆老师的团队ISIP（智能空间信息处理与分析）团队（向师门各位大佬学习）制作的一个旨在对时空数据进行管理、分析、挖掘与可视化的一体化平台，平台将在交通等领域对城市进行分析，为城市决策辅助。

平台的设计和开发主要是我和师门的两个女生共同来完成的。平台已经在外网发布，但是目前权限还没开放，有想交流的同学可以邮件（gyz@whu.edu.cu）与我们联系。

基本架构

平台最重要的部分为算法部分，在架构上没花太多功夫，与普通网站类似。由于部分数据过大，这部分数据存在MongoDB数据库中，当然也利用了MongoDB的空间索引来加速查询。
后台：SSM
前台：HTML + CSS + JS + Vue
数据库：MySQL + MongoDB
大数据：Hadoop + Spark(Livy)

功能模块

轨迹数据管理

数据下载

通过该平台，对收集的所有轨迹数据（主要为出租车的轨迹数据）进行统一管理，平台为对原始数据进行统一的预处理（去噪等）。师门的成员可以直接从平台上下载这些数据。

数据统计

平台会对所有的数据做一些统计分析，团队成员可以看到目前数据的完整性的情况，以及数据本身的一些分布规律，如以下面的一些。

场景应用模块

主要指将团队的算法研究成果应用到城市的分析当中，将理论应用与实践。

城市热点提取模块

该模块利用数据场理论，结合时空聚类等算法，对城市内的热点区域进行提取，热点区域是指在某时间段内，聚集现象明显的城市区域。

异常轨迹发现模块

该模块主要是利用一些机器学习的方法，对城市中的一些“异常轨迹”进行发现，对一些出租车的异常模式，如“疑似宰客”现象（路程长且时间长），“疑似老司机抄近路”现象（路程短且时间短）进行识别，最后进行可视化。

城市拥堵分析

利用缓速分析等方法，结合时空分析算法，对不同时段城市不同区域的拥堵情况进行计算，并可视化。

算法包模块

平台将一系列的机器学习（如谱聚类和各种分类模型等）和空间分析算法发成REST风格的服务，部分可并行计算的任务也通过Livy发成Spark的REST服务，团队成员通过界面或代码直接调用即可，一起共享服务器的优越计算性能。

后续

后面我们还讲对整个平台做持续的优化，包括增加功能，改bug等。例如把团队在遥感图像方面的研究和成果（传统方法和深度学习等）加入到整个平台当中。
