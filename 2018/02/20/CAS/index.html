Java与JVM：（四）并发的基石——CAS操作

发表于 2018-02-20 | 分类于 Java

背景

Synchronized是众所周知的锁策略，而Synchronized是一种悲观锁策略，在吞吐量上可能会有局限性（特别是早期的Synchronized，JDK1.6的偏向锁和轻量级锁为Synchronized带来了巨大的改进，这些更新同样依赖于CAS）。乐观锁则一般利用CAS来实现，在Java中位于java.util.concurrent包中，CAS可以说是撑起了一个concurrent包，足见其重要性。

CAS的全称概括了它的过程——Compare And Swap，整个操作主要包括三个值：当前内存某位置的值V，对V的预期值A，对V的欲更新值B。整个过程可以概括为“我认为V的值应该是A，如果V的确是A，就将V更新为B；但如果预期错误，V不为A，则不更新V，无论如何，结束后将V的旧值返回给我”。就是这么简单的逻辑，构筑了并发包的基础。

硬件支持

CAS之所以如此有效是因为它是一个原子操作，现在的CPU架构均支持CAS原子操作，在x86中，这个CPU指令是cmpxchg指令，这可以在阅读JNI调用的C++源码中看到（Atomic::cmpxchg）：
